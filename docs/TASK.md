# 정몽주 AR 역사탐방 — MVP 완성 작업 지시서

## 0. 한 줄 목표
기존 단일 HTML 프로토타입(`reference/prototype.html`)을 이어받아, 미구현 5개 기능을 추가해 현장 실증 가능한 MVP로 완성한다.

## 1. 맥락 (왜 이 작업인가)
- 용인시 「2026 첨단기술 융복합 실증 지원」 사업 제안용 실증 MVP.
- 핵심 컨셉: 학생이 정몽주 묘 현장에서 QR로 접속 → 안내판 인식 시 정몽주 소환 → AI 대화 + 사진 미션(비주얼 퀘스트) → 현장 이탈 시 잠금.
- **비주얼 퀘스트가 제안서의 핵심 차별점인데 프로토타입에 아직 없음.** 이번 작업의 최우선.

## 2. 이미 구현된 것 (건드리지 말 것, 회귀 금지)
- WebAR 인물 소환 (MindAR 이미지 타겟, `onTargetFound`/`onTargetLost`)
- Claude API 대화 (포은 페르소나 + 사료 근거 시스템 프롬프트 + 데모 폴백)
- 한국어 TTS (Web Speech API)
- GPS 지오펜스 (입장 검증 + `watchPosition` 이탈 잠금 + 세션 타이머/카운트다운)
- 카메라/AR 실패 시 미리보기 모드 폴백

## 3. 구현할 5개 기능 (우선순위 순)

### P1. 비주얼 퀘스트 — 사진 미션 (최우선)
인물이 미션 제시 → 학생이 유물 촬영 → 인증 → 해설 해금 → 다음 미션.
- 미션 데이터 구조: `{id, prompt, targetType, hint[], unlockText}`
- 인증 2방식:
  - `imageTarget`: 사전 등록된 유물(.mind) 인식 시 통과 (무비용)
  - `aiVision`: 촬영 사진을 Claude 비전 API에 전송해 판정 (자유 미션용, 호출 제한)
- 실패 시 `hint[]`를 단계적으로 제공 (정몽주 말투). 3회 실패 시 정답 위치 안내.
- 미션 완료 시 `unlockText`(유물 해설 + 인물 코멘트)를 대화창+말풍선+TTS로 출력.
- **실패조건**: 미션 인증 없이 다음 미션이 열리면 실패. AI 비전 호출이 미션당 3회를 초과하면 실패(비용).

### P2. AI 비전 사진 판정 엔진
P1의 `aiVision` 방식 구현.
- 촬영 → canvas로 캡처 → base64 → Claude `messages` API에 image 블록으로 전송.
- 시스템 프롬프트: "이 사진에 [target]이 있는지 yes/no와 한 줄 이유만 JSON으로". 
- 응답을 안전 파싱(JSON.parse, 실패 시 텍스트 fallback).
- **실패조건**: 키 없을 때 앱이 죽으면 실패 → 데모 모드는 항상 통과 처리.

### P3. 백엔드 서버 (API 키 보호 + 거리 재검증)
현재 브라우저 직접 호출을 서버 경유로 전환.
- 최소 Node/Express(또는 동등) 서버: `/api/chat`, `/api/vision`, `/api/verify-location`.
- API 키는 서버 환경변수(`ANTHROPIC_API_KEY`)에만 둠. 클라이언트 노출 0.
- `/api/verify-location`: 클라이언트가 보낸 좌표를 서버에서 다시 거리 계산해 통과/차단 (클라 조작 방지).
- **실패조건**: 키가 클라이언트 번들·네트워크 응답에 노출되면 실패.

### P4. 실증 데이터 로그 수집
공고 정량지표(체험 인원·이용시간·만족도)를 자동 수집.
- 세션 시작/종료, 미션 완료 수, 체류시간을 서버에 기록.
- 종료 시 간이 만족도(별점 1~5) 1문항 수집.
- 저장: 최소 JSON 파일 또는 경량 DB(SQLite 등). 과설계 금지.
- 간단한 집계 엔드포인트 `/api/stats` (총 인원·평균 체류·평균 만족도).
- **실패조건**: 개인식별정보(이름·연락처) 저장 시 실패. 익명 세션ID만.

### P5. 회전 QR 동적 토큰 (현장 캡처 우회 방지)
- 현장 키오스크/교사 화면이 일정 주기(예: 3분)로 갱신되는 QR 표시.
- QR에 단기 만료 토큰 포함 → 서버가 토큰 유효성+발급시각 검증.
- 만료 토큰으로 접속 시 차단.
- **실패조건**: 토큰이 만료돼도 입장되면 실패. (P3 서버 의존 — P3 이후 착수)

## 4. 기술 제약 (반드시 지킬 것)
- 학생 단말 = 앱 설치 0. 브라우저(WebAR)만. 헤드셋 금지.
- iOS Safari 우선 호환: 지면배치(WebXR) 의존 금지, 이미지 타겟 방식 우선.
- AR 안 되는 기기 → 채팅+미션만으로도 폴백 동작해야 함 (AR은 비주얼 레이어).
- 비용: AI 비전은 핵심 유물 아닌 자유 미션에만. 호출 상한·캐싱 필수.

## 5. 산출물
- `index.html` (또는 분리된 프론트) — 기존 프로토타입 확장
- `server/` — P3~P5 백엔드
- `missions.json` — 정몽주 묘 미션 데이터 (샘플 3개 이상)
- `README.md` — 로컬 실행법, 현장 배포 시 교체 지점(좌표·.mind·키)

## 6. 진행 순서 (단계별 검증)
1. P1+P2를 데모 모드(키 없이)로 먼저 동작시켜 흐름 검증
2. P3 서버 붙이고 키를 서버로 이전, 프론트는 서버 호출로 전환
3. P4 로그 → P5 토큰 순
각 단계마다 기존 기능(소환·대화·TTS·지오펜스) 회귀 없는지 확인 후 다음 단계.

## 7. 미확정 값 (사용자 확인 필요 — 임의로 정하지 말 것)
- 정몽주 묘 정확한 좌표·허용 반경 (현재 테스트 좌표)
- 안내판 타겟 이미지 (.mind 컴파일 대상)
- 세션 시간·QR 갱신 주기·AI 비전 호출 상한의 구체 수치
